# โจทย์ที่ 6: PATCH purchase (3 คะแนน)

แก้ไขข้อมูลคำสั่งซื้อบางส่วนตาม `id`

**Path:** `PATCH /purchases/:id`

**ไฟล์ที่ต้องสร้าง/แก้ไข:**

- `src/purchases/dto/update-purchase.dto.ts` (class `UpdatePurchaseDto`)
- `src/purchases/purchase.service.ts` (เพิ่ม method `update(id: number, dto: UpdatePurchaseDto)`)
- `src/purchases/purchase.controller.ts` (เพิ่ม endpoint `PATCH /purchases/:id`)

**เงื่อนไข:**

- ใช้ DTO + `ValidationPipe` สำหรับการรับข้อมูล
- ถ้าส่ง `items` มา → ต้องดึง `price` จาก `data/products.json` ตาม `productId` แล้วคำนวณ `totalPrice` ใหม่
- ถ้าส่งแค่ `customerName` หรือ `purchaseDate` → แค่อัปเดต field นั้น
- ถ้าไม่พบ `id` → ตอบ error response
- เขียนข้อมูลกลับไฟล์ `data/purchases.json`
- Response ต้องเป็นรูปแบบ `ApiResponse<T>`

### ตัวอย่าง Request (อัปเดตบาง field):

```json
{
  "customerName": "Johnathan Doe",
  "items": [{ "productId": 1, "quantity": 3 }]
}
```

### ตัวอย่าง Response (อัปเดตสำเร็จ):

```json
{
  "success": true,
  "data": {
    "id": 1,
    "customerName": "Johnathan Doe",
    "purchaseDate": "2024-01-15",
    "items": [{ "productId": 1, "quantity": 3, "price": 25000 }],
    "totalPrice": 75000
  },
  "message": "Updated purchase successfully"
}
```

### ตัวอย่าง Response (ไม่ผ่าน DTO validation):

```json
{
  "success": false,
  "data": null,
  "message": "Validation failed: quantity must be greater than 0"
}
```

**คะแนน:**

- ใช้ DTO + ValidationPipe ตรวจข้อมูล update: **1 คะแนน**
- อัปเดต purchase สำเร็จ และเขียนกลับไฟล์: **2 คะแนน**

**ข้อห้าม:**

1. ห้ามใช้ `process.exit()` มิเช่นนั้นจะตัดคะแนนเป็น 0 ทันที
2. ห้ามแก้ไข file `products.json` หรือ `purchases.json` หากใช้เพื่อทดสอบเสร็จแล้วให้แก้ไขเป็นข้อมูลเดิมหรืออย่า push ขึ้น git repository.
