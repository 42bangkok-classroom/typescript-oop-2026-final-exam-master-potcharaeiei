# โจทย์ที่ 5: POST purchase (3 คะแนน)

สร้างคำสั่งซื้อใหม่จากข้อมูลที่ส่งมา (ใช้ DTO + Validation)

**Path:** `POST /purchases`

**ไฟล์ที่ต้องสร้าง/แก้ไข:**

- `src/purchases/dto/create-purchase.dto.ts` (class `CreatePurchaseDto`, `PurchaseItemDto`)
- `src/purchases/purchase.service.ts` (เพิ่ม method `create(dto: CreatePurchaseDto)`)
- `src/purchases/purchase.controller.ts` (เพิ่ม endpoint `POST /purchases`)
- `src/main.ts` (เพิ่ม `ValidationPipe` global)

**เงื่อนไข:**

- ใช้ DTO + `ValidationPipe` สำหรับการรับข้อมูลจาก client
- ใช้ `class-validator` + `class-transformer` สำหรับ validation
- `id` ต้อง auto-increment จากรายการสุดท้ายใน data
- `price` ใน items ต้องดึงจาก `data/products.json` ตาม `productId`
- `totalPrice` ต้องคำนวณจาก `items` (sum of `price * quantity`)
- เขียนข้อมูลใหม่ลงไฟล์ `data/purchases.json`
- Response ต้องเป็นรูปแบบ `ApiResponse<T>`

### ตัวอย่าง Request (ข้อมูลถูกต้อง):

```json
{
  "customerName": "Alice",
  "purchaseDate": "2024-01-20",
  "items": [
    { "productId": 1, "quantity": 1 },
    { "productId": 2, "quantity": 3 }
  ]
}
```

### ตัวอย่าง Response (สร้างสำเร็จ):

```json
{
  "success": true,
  "data": {
    "id": 6,
    "customerName": "Alice",
    "purchaseDate": "2024-01-20",
    "items": [
      { "productId": 1, "quantity": 1, "price": 25000 },
      { "productId": 2, "quantity": 3, "price": 500 }
    ],
    "totalPrice": 26500
  },
  "message": "Created purchase successfully"
}
```

### ตัวอย่าง Request (ไม่ผ่าน DTO validation):

```json
{
  "customerName": "",
  "items": []
}
```

### ตัวอย่าง Response (Bad Request):

```json
{
  "success": false,
  "data": null,
  "message": "Validation failed: customerName should not be empty, items must contain at least 1 item"
}
```

**คะแนน:**

- ใช้ DTO + ValidationPipe ตรวจข้อมูล: **1 คะแนน**
- สร้าง purchase สำเร็จ, คำนวณ `totalPrice`, เขียนกลับไฟล์: **2 คะแนน**

**ข้อห้าม:**

1. ห้ามใช้ `process.exit()` มิเช่นนั้นจะตัดคะแนนเป็น 0 ทันที
2. ห้ามแก้ไข file `products.json` หรือ `purchases.json` หากใช้เพื่อทดสอบเสร็จแล้วให้แก้ไขเป็นข้อมูลเดิมหรืออย่า push ขึ้น git repository.
