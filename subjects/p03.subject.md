# โจทย์ที่ 3: GET all purchases with filter (2 คะแนน)

เพิ่มความสามารถในการกรองรายการคำสั่งซื้อด้วย query parameters

**Path:** `GET /purchases`

**ไฟล์ที่ต้องแก้ไข (จากข้อ 2):**

- `src/purchases/purchase.service.ts` (แก้ไข method `findAll()` ให้รับ query parameters)
- `src/purchases/purchase.controller.ts` (รับ query parameters ด้วย `@Query()`)

**เงื่อนไข:**

- Filter ด้วย `customerName` (substring match, case-insensitive)
- Filter ด้วยช่วงวันที่ (`startDate`, `endDate`) - กรองตาม `purchaseDate`
- สามารถใช้ filter หลายตัวพร้อมกันได้
- ถ้าไม่มี filter ให้คืนทั้งหมด (เหมือนข้อ 2)
- message ต้องเป็น `"Filtered purchases successfully"` เมื่อมี filter

### ตัวอย่างที่ 1 – Filter ด้วย `customerName`

**Request:**

```text
GET /purchases?customerName=John
```

**Response:**

```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "customerName": "John Doe",
      "purchaseDate": "2024-01-15",
      "items": [{ "productId": 1, "quantity": 2, "price": 25000 }],
      "totalPrice": 50000
    }
  ],
  "message": "Filtered purchases successfully"
}
```

### ตัวอย่างที่ 2 – Filter ด้วยช่วงวันที่

**Request:**

```text
GET /purchases?startDate=2024-01-15&endDate=2024-01-17
```

**Response:**

```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "customerName": "John Doe",
      "purchaseDate": "2024-01-15",
      "items": [],
      "totalPrice": 0
    },
    {
      "id": 2,
      "customerName": "Jane Smith",
      "purchaseDate": "2024-01-16",
      "items": [],
      "totalPrice": 0
    }
  ],
  "message": "Filtered purchases successfully"
}
```

**ข้อห้าม:**

1. ห้ามใช้ `process.exit()` มิเช่นนั้นจะตัดคะแนนเป็น 0 ทันที
2. ห้ามแก้ไข file `products.json` หรือ `purchases.json` หากใช้เพื่อทดสอบเสร็จแล้วให้แก้ไขเป็นข้อมูลเดิมหรืออย่า push ขึ้น git repository.
